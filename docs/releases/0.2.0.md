# yoq 0.2.0

`0.2.0` is the release where the app-first model became the default story of the project.

The main changes in this cycle:

- app-first local and remote control flows now line up around app apply, status, history, rollback, and summaries
- releases carry richer rollout state, rollout control state, checkpoints, and recovery metadata
- mixed workloads now travel through the same app snapshot model: services, workers, crons, and training jobs
- cluster and local rollout behavior both gained clearer readiness gating, rollback behavior, and operator-facing rollout views
- dedicated operator and network smoke lanes now exist in the repo and in CI

Operationally, this release is a lot more coherent than the earlier `0.1.x` line. The app model, release model, and rollout model now fit together instead of feeling like adjacent features.

Highlights:

## app-first control plane

- canonical app snapshots and release plans are now the center of local and remote apply flows
- `yoq apps`, `yoq status --app`, `yoq history --app`, and `yoq rollback --app` now share the same underlying release data
- rollback defaults are clearer and previous successful release tracking is exposed directly

## workload parity

- app snapshots now carry services, workers, crons, and training definitions together
- remote worker runs and training controls are app-scoped
- cluster training logs can be proxied from the hosting agent

## rollout behavior

- rollout strategies are preserved in the manifest and release snapshot
- rollout state, control state, failure details, targets, and checkpoints are surfaced in status and history views
- pause, resume, cancel, and recovery flows are available for active rollouts

## reliability work

- operator smoke coverage now has a dedicated build target and CI jobs
- network rollout status and metrics coverage no longer depends on placeholder skips
- recent fast-suite regressions in the clustered training-log path and shared in-memory test DB handling were fixed before this release

## notes

- local prerelease checks still skip the known slow compression-related layer tests through `YOQ_SKIP_SLOW_TESTS=1`
- the release workflow remains the publishing gate for release artifacts
